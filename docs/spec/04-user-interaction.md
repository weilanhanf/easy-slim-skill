# Step 4: 用户交互

## 目标

定义用户交互流程，包括引导式问答、图片识别交互、进度查询等。

## 前置条件

- 完成 [03-skill-definition.md](03-skill-definition.md)

## 任务清单

请选择你的性别：
1. 男
2. 女

1. 温柔鼓励型 💗

### 4.2 图片识别交互

**成功场景**:

```
用户: [发送食物图片]
系统: 🍽️ 已识别你的午餐

| 食物 | 份量 | 热量 |
|------|------|------|
| 白米饭 | 150g | 195 kcal |
| 红烧肉 | 80g | 280 kcal |
| 清炒青菜 | 100g | 25 kcal |

📊 今日汇总
- 本次摄入: 500 kcal
- 今日已摄入: 500 kcal
- 剩余额度: 1290 kcal ✓

💡 建议: 午餐热量适中，晚餐建议控制在 600kcal 以内。
```

**需要确认场景**:

```
用户: [发送模糊图片]
系统: 🤔 我看到可能是以下食物，请确认：

1. 炒饭（约 500kcal）
2. 炒面（约 550kcal）
3. 其他（请手动输入）

回复数字选择，或告诉我具体是什么～
用户: 1

系统: ✅ 已记录炒饭 500kcal
今日已摄入 500kcal，剩余 1290kcal
```

**失败场景**:

```
用户: [发送非食物图片]
系统: 这看起来不是食物照片哦 😄

如果你已经吃饭了，可以：
1. 重新拍一张食物照片
2. 手动告诉我吃了什么，如："米饭一碗，红烧肉几块"
```

## 交互状态机

```
┌─────────────┐
│   空闲状态   │
└──────┬──────┘
       │ 用户消息
       ▼
┌─────────────┐
│ 唤醒词检测  │  ← 检查是否包含"胖虎"
└──────┬──────┘
       │
       ├── 不包含胖虎 ──→ 不触发技能
       │
       └── 包含胖虎 ──┐
                      ▼
              ┌─────────────┐
              │   意图识别   │
              └──────┬──────┘
                     │
                     ├── 图片消息 ──→ 图片识别流程
                     ├── 减肥/计划 ──→ 检查初始化状态
                     ├── 吃了/记录 ──→ 记录流程
                     ├── 进度/查看 ──→ 进度报告
                     ├── 设置/修改 ──→ 设置流程
                     └── 只有胖虎 ──→ 返回帮助提示
```

## 边界情况处理

| 情况 | 处理方式 |
|------|----------|
| 用户输入无效数值 | 提示范围，要求重新输入 |
| 用户中途取消 | 保存当前进度，提示可继续 |
| 连续多张图片 | 依次处理，汇总显示 |
| 语音消息 | 提示转为文字或提供转写服务 |

## 验证标准

- [ ] 所有交互流程定义清晰
- [ ] 边界情况覆盖完整
- [ ] 响应消息友好易懂
- [ ] 状态机逻辑正确
